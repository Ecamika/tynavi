# 2026-06-08 变更记录

## [feat] 为 tungstenite 提供 Selector 支持

### 变更内容
- 新增 `tungstenite` feature flag，引入 `tungstenite = "0.29.0"` 作为可选依赖。
- 为 `tungstenite::protocol::Message` 提供 `AsSelector` 入口与完整类型检查过滤方法：`is_text`、`is_binary`、`is_ping`、`is_pong`、`is_close`、`is_frame` 及其 `not_*`/`cond_*` 变体。
- 为 `Message` 提供路由方法：`as_text`、`as_binary`、`as_ping`、`as_pong`、`as_close`，分别路由到 `Utf8Bytes`、`Bytes`、`Option<CloseFrame>` 等载荷类型。
- 为 `Message` 提供 `empty`、`not_empty` 及其 `cond_*` 变体，复用 `Message::is_empty()` 语义。
- 为 `tungstenite::protocol::frame::Utf8Bytes` 提供 `AsSelector` 入口与全套字符串过滤方法：`starts_with`、`ends_with`、`contains`、`empty`、`contains_char` 及其 `not_*`/`cond_*` 变体。
- 为 `tungstenite::protocol::frame::CloseFrame` 提供 `AsSelector` 入口与路由方法：`code()` → `CloseCode`、`reason()` → `Utf8Bytes`。
- 为 `tungstenite::protocol::frame::coding::CloseCode` 提供 `AsSelector` 入口与 `is_allowed`、`not_is_allowed` 及其 `cond_*` 变体。
- 为 `tungstenite::Bytes` 提供 `AsSelector` 入口与 `empty`、`not_empty` 及其 `cond_*` 变体。
- 新增 `tests/tungstenite_selector.rs` 集成测试，覆盖类型检查、路由链式调用、条件变体、`empty` 判断及 `CloseFrame`/`CloseCode`/`Bytes` 场景。

### 涉及文件
- `Cargo.toml` [Changed] 新增 `tungstenite` 可选依赖、`tungstenite` feature 定义，并将 `tungstenite` 纳入 `full` 聚合 feature。
- `src/selector/extension.rs` [Changed] 条件编译导出 `tungstenite` 模块。
- `src/selector/extension/tungstenite/mod.rs` [Added] 实现 `Message`、`Utf8Bytes`、`CloseFrame`、`CloseCode`、`Bytes` 的 Selector 扩展。
- `tests/tungstenite_selector.rs` [Added] 集成测试，覆盖全部新增 API。
- `changelogs/2026-06-08.md` [Added] 记录本次变更。

### 影响
- 启用 `tungstenite` feature 后，调用方可以直接对 WebSocket `Message`、`CloseFrame`、`Utf8Bytes` 等类型使用 Selector 链式导航与过滤。
- `tungstenite` 被纳入 `full` feature，默认启用；使用 `--no-default-features` 仍可完全剥离外部依赖，保持零依赖能力。
- `Utf8Bytes` 未提供 `as_str` 路由方法，因其底层为 `Deref<Target = str>` 的 newtype，`str` 不满足 `Selector` 的 `Sized` 约束；字符串过滤方法直接在 `Utf8Bytes` Selector 上提供，语义等价。

### 原因
- 将 Selector 模式扩展到 tungstenite WebSocket 类型，满足对 WebSocket 消息进行不可变导航与过滤的常见需求。
