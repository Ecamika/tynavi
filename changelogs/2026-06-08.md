# 2026-06-08 变更记录

## [feat] 为 http 和 axum crate 添加 Selector 支持

### 变更内容
- 在 `Cargo.toml` 中新增 `http = "1.0"` 和 `axum = "0.8"` 作为可选依赖
- 新增 `http` 和 `axum` 两个 feature flag，并纳入 `full` feature
- 为 `http` crate 的以下类型实现 `AsSelector` 和 `Selector` 固有方法：
  - `http::Request<B>`：提供 `method()`、`uri()`、`version()`、`headers()`、`body()` 路由方法
  - `http::Response<B>`：提供 `status()`、`version()`、`headers()`、`body()` 路由方法
  - `http::Method`：提供 `AsSelector` 入口，复用已有 `eq`/`ord` trait 进行比较过滤
  - `http::StatusCode`：提供 `is_informational`、`is_success`、`is_redirection`、`is_client_error`、`is_server_error` 分类过滤及完整变体体系，复用 `eq`/`ord` trait
  - `http::Version`：提供 `AsSelector` 入口，复用 `eq` trait
  - `http::Uri`：提供 `starts_with`、`ends_with`、`contains`、`empty` 字符串过滤及完整变体体系，提供 `path()`、`query()`、`host()`、`port()`、`scheme()` 路由方法
  - `http::HeaderMap<V>`：提供 `empty`、`contains_key` 过滤及完整变体体系，提供 `keyof(name)` 路由方法
  - `http::header::HeaderValue`：提供 `starts_with`、`ends_with`、`contains`、`empty` 字符串过滤及完整变体体系（非 UTF-8 值安全处理为不匹配）
- 为 `axum` crate 的以下提取器实现 `AsSelector` 和 `inner()` 路由方法：
  - `axum::extract::Path<T>`
  - `axum::extract::Query<T>`
  - `axum::extract::Json<T>`
  - `axum::extract::State<T>`
- 新增 `tests/http_selector.rs` 集成测试，覆盖 Request/Response 路由链、Method/StatusCode/Version 比较、Uri 字符串过滤和路由、HeaderMap 操作、HeaderValue 字符串过滤、复杂链式调用
- 新增 `tests/axum_selector.rs` 集成测试，覆盖各提取器的 `inner()` 路由和链式调用

### 涉及文件
- `Cargo.toml` [Changed] 新增 `http` 和 `axum` 可选依赖，新增 `http`/`axum` feature，更新 `full` feature 列表
- `src/selector.rs` [Changed] `Selector` 的 `Current` 泛型参数扩展为 `Current: ?Sized`，`route_to` 的 `R` 泛型参数扩展为 `R: ?Sized`，`with` 和 `same_parent` 的泛型参数同步扩展，支持 `str` 等 DST 类型作为 Selector 的游标类型
- `src/traits.rs` [Changed] `AsSelector` trait 的 `C` 泛型参数扩展为 `C: ?Sized`
- `src/selector/extension.rs` [Changed] 注册 `http` 和 `axum` 模块
- `src/selector/string.rs` [Changed] 为 `Selector<'a, str, P>` 新增完整的字符串过滤方法（`starts_with`、`ends_with`、`contains`、`empty`、`contains_char` 及完整变体体系）
- `src/selector/extension/http/mod.rs` [Added] http 类型 Selector 实现
- `src/selector/extension/axum/mod.rs` [Added] axum 提取器 Selector 实现
- `tests/http_selector.rs` [Added] http 集成测试
- `tests/axum_selector.rs` [Added] axum 集成测试

### 影响
- 启用 `http` 或 `axum` feature 后，用户可通过 `.as_selector()` 对 HTTP 请求/响应、URI、Header、axum 提取器进行不可变链式导航和过滤
- 不启用对应 feature 时无影响，保持零依赖

### 原因
- 扩展 tynavi 的 Selector 模式到 Web 开发中最常用的 HTTP 类型，使 axum handler 中能以声明式链式 API 检查和路由请求数据
