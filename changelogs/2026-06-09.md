# 2026-06-09 变更记录

## [feat] 新增 reqwest 扩展支持

### 变更内容
- 新增可选依赖与 `reqwest` feature，并将其纳入默认 `full` 功能集
- 新增 `selector::extension::reqwest` 模块，为 `reqwest::Request`、`reqwest::Response`、`reqwest::Url`、`reqwest::Body` 提供 Selector 扩展
- 为 `Request` 和 `Response` 补充路由、状态过滤、版本比较与 `content_length_*` 过滤能力
- 为 `Url` 补充路径、查询、主机、协议与端口相关导航和过滤方法
- 为 `Body` 在可共享字节场景下补充空值、前后缀与包含关系过滤
- 新增 `reqwest` 集成测试并在 README 中补充功能说明与示例

### 涉及文件
- `Cargo.toml` [Changed] 新增 `reqwest` 可选依赖、feature 开关，并将其加入 `full`
- `src/selector/extension.rs` [Changed] 注册 `reqwest` 扩展模块导出
- `src/selector/extension/reqwest/mod.rs` [Added] 新增 `reqwest` 相关 Selector 扩展实现
- `tests/reqwest_selector.rs` [Added] 新增 `reqwest` 扩展的路由、过滤与回溯测试
- `README.md` [Changed] 补充 `reqwest` 功能说明与使用示例
- `changelogs/2026-06-09.md` [Added] 记录本次 `reqwest` 扩展变更

### 影响
- 调用方可直接对 `reqwest` 请求、响应、URL 和可重用 Body 进行链式导航与过滤
- `reqwest` feature 会联动启用现有 `http` 扩展，使 `headers()` 等返回值可继续复用已有 HTTP Selector 能力
- 仍不支持 `reqwest::Response` body 的只读路由，避免在当前共享借用模型下暴露不安全的消费式接口

### 原因
- 现有扩展已覆盖底层 `http`、`axum`、`tungstenite`、`serde_json`，但缺少常见 HTTP 客户端 `reqwest` 的直接支持
- 通过补齐 `reqwest` 扩展，可以让网络请求对象在业务代码里也复用统一的 Selector 导航与过滤体验
