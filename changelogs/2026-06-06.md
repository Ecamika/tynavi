# 2026-06-06 变更记录

## [feat][docs][chore] 扩展 Selector 类型支持并补充项目元数据与本地技能文件

### 变更内容
- 为 `Selector` 新增 `Option<T>` 与 `Result<T, E>` 相关扩展，支持从 `Option` 中 `flatten()` 到内部值，以及从 `Result` 中提取 `ok()` / `err()` 分支。
- 为字符串选择器补充 `not_starts_with`、`not_ends_with`、`not_contains` 及对应 `cond_not_*` 方法，使字符串筛选能力与数字选择器的正反向比较接口更一致。
- 统一整理核心 `Selector`、数值选择器与基础 trait 的实现格式，并在 `Cargo.toml` 中补充 crate 描述信息。
- 新增仓库内本地 agent skill 文件，补充 `git-commit`、`plan-mode` 与 `skill-creator` 相关定义、脚本和参考资源。

### 影响
- 调用方现在可以直接对 `Option` 和 `Result` 建立选择器链，并对字符串执行与数字接口风格一致的反向条件筛选。
- 这批改动主要扩展 API 能力并整理实现形式，不引入已知破坏性接口变更；新增 `.agents/` 文件会使仓库包含本地辅助技能资源。

### 原因
- 为了让 `Selector` 在常见容器类型与字符串场景下具备更完整的一致性接口，减少调用方手写拆分逻辑。
- 同时补充包描述信息和本地技能资源，方便仓库发布说明与后续协作流程复用。
