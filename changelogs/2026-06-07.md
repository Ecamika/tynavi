## [feat] 补全 HashMap Selector 的集合访问与过滤方法

### 变更内容
- 为 `HashMap` Selector 新增 `empty`、`not_empty`、`keyof`、`get`、`find_key`、`find` 等访问方法。
- 为 `HashMap` Selector 新增 `any`、`not_any`、`all`、`not_all`、`none` 以及 `contains_key`、`contains_value` 等集合过滤方法。
- 为新增方法补齐 `cond_*` 条件变体与异步谓词过滤变体，保持与切片 Selector 近似的链式 API 风格。

### 涉及文件
- `src/selector.rs` [Changed] 导出 `hash_map` Selector 模块，使新实现可从主 selector 模块访问。
- `src/selector/hash_map.rs` [Changed] 新增 `HashMap` 的 `AsSelector` 实现以及键值访问、谓词过滤、条件变体和异步变体方法。
- `changelogs/2026-06-07.md` [Changed] 追加本次 HashMap Selector 功能补全记录。

### 影响
- 调用方可以直接对 `HashMap` Selector 进行键查找、键值筛选和集合状态判断，无需手写底层 `filter`/`route_to` 逻辑。
- `contains_key`/`get`/`keyof` 支持借用键查询，`contains_value` 相关方法要求值类型实现 `PartialEq`。

### 原因
- 补齐 `HashMap` Selector 与现有切片 Selector 相同层级的常用集合操作能力，降低不同容器类型之间的使用心智切换成本。

## [feat] 为 `serde_json::Value` 与 `Map<String, Value>` 提供 Selector 支持

### 变更内容
- 为 `serde_json::Value` 新增 `AsSelector` 入口与完整类型检查方法（`is_null`、`is_bool`、`is_number`、`is_string`、`is_array`、`is_object` 及其 `not_*`/`cond_*` 变体）。
- 为 `Value` 新增路由方法 `as_null`、`as_bool`、`as_number`、`as_str`、`as_object`，支持路由到对应子类型并复用已有 Selector 扩展。
- 为 `Value` 新增快捷导航方法 `keyof`、`indexof`、`first`、`last`，直接从 Object/Array 中提取子值。
- 为 `Value` 新增通用过滤方法 `empty`、`not_empty` 及其 `cond_*` 变体，支持 Null、String、Array、Object 的空值判断。
- 为 `serde_json::Map<String, Value>` 新增 `AsSelector` 入口与 `empty`、`not_empty`、`keyof`、`contains_key` 等方法及其条件变体。

### 涉及文件
- `src/selector/extension/serde_json/mod.rs` [Changed] 新增 `serde_json::Value` 的 `AsSelector` 实现与全套类型检查、路由、导航、过滤方法。
- `src/selector/extension/serde_json/map.rs` [Added] 新增 `serde_json::Map<String, Value>` 的 `AsSelector` 实现与键访问、空值判断、条件变体方法。
- `tests/serde_json_selector.rs` [Added] 新增针对 `Value` 与 `Map` Selector 的完整功能测试，覆盖类型检查、路由、导航、空值判断及链式组合场景。
- `changelogs/2026-06-07.md` [Changed] 追加本次 serde_json Selector 支持记录。

### 影响
- 启用 `serde_json` feature 后，调用方可以直接对 `serde_json::Value` 与 `Map<String, Value>` 使用 Selector 链式导航与过滤。
- `as_str` 路由到 `String` 类型以复用现有字符串 Selector 扩展；`as_array` 因 `route_to` 的 `Sized` 约束未在本次提供，Array 操作通过 `Value` 层面的 `first`/`indexof`/`last` 直接支持。

### 原因
- 将 Selector 模式扩展到 serde_json 数据类型，满足对动态 JSON 结构进行不可变导航与过滤的常见需求。

## [docs] 补充 Selector 方法语义一致性原则到 AGENTS.md

### 变更内容
- 在 `AGENTS.md` 的「Selector API 规范」章节中新增「语义一致性原则」小节。
- 明确 Selector 方法命名**不应直接照搬底层容器的原始方法名**，而应体现 Selector 的导航与过滤语义。
- 给出典型示例对照表：`Vec::get` → `indexof`、`HashMap::get` → `keyof`、`Option::unwrap` → `flatten`、`Result::ok()` → `ok()`。

### 涉及文件
- `AGENTS.md` [Changed] 在「方法命名与变体规范」前插入「语义一致性原则」小节，提供底层 API 与 Selector 语义命名的对照表。

### 影响
- 后续新增 Selector 扩展时，方法命名需遵循语义一致性原则，避免直接复制底层容器的方法名。
- 现有代码中已使用 `indexof`、`keyof`、`flatten`、`ok` 等命名，与新增规范一致，无需回改。

### 原因
- 统一 Selector 方法命名风格，强化「导航与过滤」的核心语义，避免底层容器 API 命名对 Selector API 的干扰。
